How to Use This Playbook

1. Pick the module you are implementing.
2. Open that module section only.
3. Complete work in this order: tables -> entity -> repository/HQL -> service validation -> controller -> Angular

      model/service/pages -> dashboard -> permissions -> testing.
4. Do not move to next module until the current module CRUD/workflow/dashboard/permission/testing are done.

Reusable Implementation Checklist for Every Module

5. Create all master/child/workflow tables with PK/FK/unique/audit columns.
6. Create entity classes aligned with table/column names.
7. Create repository HQL methods: save, update, getById, list/search/filter, status-based search, soft delete/status change if

      allowed.
8. Create service validations: uniqueness, existence, workflow, branch/role/permission and business rules.
9. Create controller endpoints: create, update, getById, list/search, status/workflow action, dashboard summary.
10. Create Angular model/service, dashboard page, list page, create/edit/view pages and workflow pages.
11. Add DataTable-style search/filter/pagination/export, SweetAlert confirmations, status badges and action buttons.
12. Map permissions and hide/guard routes/buttons while enforcing backend checks too.
13. Test valid cases, invalid cases, role-wise access and end-to-end workflow.
