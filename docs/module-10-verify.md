# Module 10 Verify Guide

## 1. Open the module
- Go to `/profit/dashboard`
- Open `/profit/ratios`
- Open `/profit/ratios/new`
- Open `/profit/schedules`
- Open `/profit/schedules/new`
- Open `/profit/postings`
- Open `/profit/postings/run`

## 2. Check playbook coverage
- Dashboard should show active profit ratios, pending posting cycles, posted this month, failed posting logs, current PSR table and upcoming posting run.
- Profit ratio list should support search, print, export, view, edit, create schedule, run posting and archive or restore.
- Profit ratio view should show detail, effective window, schedule list, posting history and failed cases.
- Profit schedule list should show account linkage, posting frequency, next posting date, account and ratio navigation, run posting and archive or restore.
- Profit schedule view should show current ratio and posting history.
- Profit posting run page should accept account or schedule input and return processed, posted and failed result counts.
- Profit posting list should support search, print, export, view, account navigation, schedule navigation and rerun.
- Profit posting view should show posting period, amount, snapshot balances, posted by and failure reason.

## 3. Data verification
- Active database for this project is `sbms`.
- Run `docs/module-10-manual-seed.sql` after backend startup if you want the sample dataset.
- Expected seeded counts:
  - `profit_ratio = 15`
  - `profit_schedule = 15`
  - `profit_posting = 15`
  - `balance_snapshot = 15`

## 4. Navigation verification
- Sidebar menu should now include `Profit Management` with dashboard, ratio, schedule and posting routes.
- Account list now has direct `Profit Schedule` and `Profit Postings` action buttons.
- Account view now has `Profit Schedule`, `Profit Postings` and `Profit Ratio` navigation buttons.

## 5. What to test manually
- Create one ratio from `/profit/ratios/new`.
- Create one schedule from `/profit/schedules/new`.
- Run one posting from `/profit/postings/run`.
- Open one ratio view and verify linked schedules and posting history.
- Open one posting view and verify snapshot balance, period and failure reason rendering.
- Print one list page and one view page to confirm only content prints.

## 6. Postman support
- Request samples are in `docs/module-10-postman-samples.json`.
- Seed SQL is in `docs/module-10-manual-seed.sql`.
- Balance snapshots are auto-generated by posting run; there is no separate Module 10 snapshot create API.
