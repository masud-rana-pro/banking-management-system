Process Flow Map

1. Customer To KYC To Account

Role order: OPERATIONS_OFFICER -> BRANCH_MANAGER -> OPERATIONS_OFFICER

  1. Create customer from Customer module
  2. Manage address and identity
  3. Create KYC profile and submit
  4. Verify / approve / reject / return from KYC approval flow
  5. Create account opening request
  6. Submit and review request
  7. Approve request and show final account in Account List

2. Account To Transaction To Statement

Role order: TELLER -> OPERATIONS_OFFICER

  1. Use approved account
  2. Show deposit / withdraw / transfer screens
  3. Open transaction list and one voucher/journal view
  4. Go to Statement module
  5. Create customer or branch statement request
  6. Show preview / download / export

3. Financing To Contract To Shariah

Role order: INVESTMENT_OFFICER -> SHARIAH_BOARD_MEMBER

  1. Create or open financing application
  2. Submit -> verify -> review -> approve
  3. Disburse financing
  4. Open schedule and repayment collection
  5. Generate contract from source module reference
  6. Show customer sign and shariah sign path
  7. Open shariah case review and correction queue

4. Reports To Accounting Proof

Role order: SYSTEM_ADMIN

  1. Show Report Dashboard
  2. Open Management P&L
  3. Open Expense Register
  4. Open Trial Balance and view journals
  5. Open Ledger Profit & Loss
  6. Show reconciliation, branch summary, income heads, expense heads, and drill-down
